# План: интеграция дизайна Сообщения

## Что уже есть

- **Файл макета:** [design-html/messages/code.html](design-html/messages/code.html) — полная страница: сайдбар (Hotel PMS) + основной блок (заголовок + список чатов `w-80` + панель чата).
- **Текущая реализация:**
  - [Dashboard.tsx](src/components/Dashboard.tsx): при `currentView === 'messages'` рендерит flex: `MessagesView` (flex-1) + при `selectedChatId` — `ChatPanel` (адаптивная ширина).
  - [MessagesView.tsx](src/components/MessagesView.tsx): одна колонка — заголовок, CTA Avito (если нужно), три стат-карточки (Всего / Непрочитанные / Новые), фильтры (поиск + объект + статус), список чатов карточками.
  - [ChatPanel.tsx](src/components/ChatPanel.tsx): шапка чата (аватар, имя, телефон, «Создать бронь», меню статуса), лента сообщений (без разделителей по датам), поле ввода (шаблоны кнопками сверху, скрепка, textarea, отправить).

В макете: **сайдбар не трогаем** — он уже есть в приложении ([Sidebar.tsx](src/components/Sidebar.tsx)). Переносим только контент: левая колонка списка чатов + правая панель чата.

---

## Порядок реализации (важно)

Строго соблюдать порядок, иначе сломается layout:

1. **Dashboard.tsx** — изменить layout сообщений: левая колонка фиксированной ширины под список, правая под панель чата (или плейсхолдер).
2. **MessagesView.tsx** — перевести в вид «колонки списка» под новый layout.
3. **ChatPanel.tsx** — обновить вёрстку и стили под макет.

Если менять MessagesView или ChatPanel до правок Dashboard — layout будет некорректным.

## Вопросы для уточнения

1. **Статистика и фильтры** — оставить все (карточки + фильтры), убрать карточки, или только поиск?
2. **Ширина левой колонки** — фиксировать `w-80` или гибкая?
3. **Шапка чата: «View Booking» и «Details»** — есть ли привязка чата к бронированию (Booking ID)? Реальные действия или только визуал?
4. **Лента сообщений** — разделители по датам? «Seen» и индикатор набора — заглушки или не показывать?
5. **Поле ввода** — «Use Quick Reply» = наши шаблоны под полем. Кнопку «Перевод» не добавлять.
6. **Иконки** — Lucide или Material Symbols для экрана Сообщений?
7. **Цвет primary** — использовать только `--primary` из [index.css](src/index.css). При несовпадении с макетом (`#00bda4`) обновить значение токена в index.css (все темы). Новых переменных не вводить.
8. **Пустое состояние** — плейсхолдер «Выберите чат» справа или панель скрыта?

## Предлагаемая структура работ (после ответов)

- **Порядок:** 1) Dashboard.tsx, 2) MessagesView.tsx, 3) ChatPanel.tsx.
- **index.css:** при необходимости привести `--primary` к макету `#00bda4` во всех темах. Новых переменных не вводить.
- **Layout в Dashboard:** левая колонка фиксированной ширины под список, правая — под `ChatPanel`; при отсутствии выбранного чата — плейсхолдер справа (если выберете в п.8).
- **MessagesView:** перевести в «колонку списка»: поиск (и при необходимости фильтры/карточки); список карточек чатов по макету; выделение выбранного через `border-l-4 border-primary` и фон.
- **ChatPanel:** шапка по макету (объект, Booking ID при наличии, View Booking / Details, more_vert); лента с разделителями дат и стилями пузырей; поле ввода в одном блоке rounded-xl + ссылка «Quick Reply» (шаблоны) и подсказка Shift+Enter. Кнопку «Перевод» не добавлять.
- **Стили:** при желании класс `.custom-scrollbar` для списка/чата.
